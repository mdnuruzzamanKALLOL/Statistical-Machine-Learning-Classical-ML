# 🌀 Gaussian Mixture Models — Soft, Probabilistic Clustering

> Status: ✅ Complete — [Open the notebook →](04_gaussian_mixture_models.ipynb)

Every clustering method so far has given each point exactly one cluster label. A Gaussian Mixture Model gives each point a *probability distribution* over clusters instead — "70% likely in cluster A, 30% likely in cluster B" is a valid, meaningful GMM output. GMM also generalizes K-Means directly: K-Means turns out to be a special, restricted case of GMM (spherical clusters, hard assignment, equal weights), fit with a simpler algorithm. This notebook derives the EM algorithm that fits a GMM, then re-tests DBSCAN's varying-density failure case and K-Means' elliptical-cluster limitation to see whether GMM's added flexibility actually fixes them.

## 📖 Concept & Intuition

```
K-Means:  each point belongs to exactly ONE cluster (hard assignment)
          each cluster is a single point (centroid) + implicit equal spherical radius

GMM:      each point has a PROBABILITY of belonging to each cluster (soft assignment)
          each cluster is a full Gaussian: its own mean, its own shape (covariance), its own weight
```

## 🧮 Mathematical Explanation

### 1. The mixture model

$$p(x) = \sum_{k=1}^{K} \pi_k \, \mathcal{N}(x \mid \mu_k, \Sigma_k)$$

The data is modeled as generated by $K$ Gaussian components, each with mean $\mu_k$, covariance $\Sigma_k$, and mixing weight $\pi_k$ (with $\sum_k \pi_k = 1$). Unlike linear regression's normal equation, there's no closed-form solution, because the cluster assignments themselves are latent (unknown).

### 2. The EM algorithm

**E-step** — compute the *responsibility* (posterior probability) of each component for each point:

$$\gamma_{ik} = \frac{\pi_k \, \mathcal{N}(x_i \mid \mu_k, \Sigma_k)}{\sum_{j=1}^K \pi_j \, \mathcal{N}(x_i \mid \mu_j, \Sigma_j)}$$

**M-step** — update each component's parameters using the responsibilities as soft weights:

$$N_k = \sum_i \gamma_{ik}, \quad \mu_k = \frac{1}{N_k}\sum_i \gamma_{ik} x_i, \quad \Sigma_k = \frac{1}{N_k}\sum_i \gamma_{ik}(x_i - \mu_k)(x_i - \mu_k)^T, \quad \pi_k = \frac{N_k}{n}$$

Alternating E and M steps provably increases the data log-likelihood every iteration (§3's log-likelihood curve rises monotonically, exactly as K-Means' inertia can only fall), converging to a local optimum. In practice, $\Sigma_k$'s update needs a small regularization term (`reg_covar`) added to its diagonal — without it, a component that collapses onto very few points can produce a singular (non-invertible) covariance matrix, crashing the density calculation, as this notebook's own development process discovered directly.

### 3. K-Means as a restricted special case

Fixing every $\Sigma_k$ to the same spherical (scalar-multiple-of-identity) matrix, replacing soft responsibilities with hard winner-take-all assignment, and fixing $\pi_k$ equal for every component reduces the EM update exactly to K-Means' assign/update loop. §4 tests this relationship directly — spherical-covariance GMM's hard assignments closely match K-Means' on data K-Means already handles well.

### 4. BIC and AIC for model selection

$$\text{BIC} = -2\ln(\hat L) + p\ln(n), \qquad \text{AIC} = -2\ln(\hat L) + 2p$$

Unlike the elbow method or silhouette score (both heuristics with no direct connection to a likelihood), BIC and AIC come directly from the mixture model GMM already is, penalizing the number of free parameters $p$ against the achieved log-likelihood $\hat L$. §8 shows both turning back upward past the true component count, giving a genuinely principled stopping point.

## 📋 Covariance Type Reference

| `covariance_type` | Shape per component | Closest to |
|---|---|---|
| `spherical` | Equal variance, all directions | K-Means (§4) |
| `diag` | Different variance per axis, no rotation | Axis-aligned ellipses |
| `tied` | One shared covariance across all components | A single shape, different positions |
| `full` | Arbitrary elliptical shape and rotation per component | Maximum flexibility (best on elliptical data, §5) |

## ⚠️ Common Pitfalls & Gotchas

1. **Comparing a from-scratch implementation against sklearn from different random initializations and concluding a mismatch is a bug** — the first validation attempt in this notebook did exactly this; forcing both to start from identical parameters (§3) is the only way to isolate an algorithm error from an initialization difference (the same lesson from the K-Means topic).
2. **Forgetting covariance regularization** — a component can genuinely collapse onto too few points during EM, producing a singular covariance matrix; sklearn's `reg_covar` (and this notebook's own from-scratch fix) exists specifically to prevent this crash.
3. **Assuming GMM always outperforms simpler methods because it's more flexible** — §13 shows GMM clearly underperforming Hierarchical/DBSCAN on the crescents dataset; covariance flexibility helps with cluster *shape*, not with tracing thin, curved, chain-like structures.
4. **Using `spherical` or `diag` covariance on genuinely elliptical, rotated data** — §5 shows `full` covariance reaching ARI=0.987 versus much lower scores for the more restricted covariance types on the same rotated-ellipse data.
5. **Trusting a single EM run without multiple restarts (`n_init`)** — like K-Means, EM only guarantees a local optimum; sklearn's default `n_init=1` should usually be raised for real use (this notebook uses `n_init=10` throughout its comparison sections).
6. **Ignoring feature scaling because covariance "should" handle it** — §11 shows real ARI degradation on unscaled data despite full covariance's theoretical ability to represent any linear scale difference; EM's practical convergence still benefits from comparable feature scales.
